Símbolos ⊢ e ∴
Os dois significam "portanto" — a diferença é só o formato de escrita.
Horizontal
⊢ — numa linha só
Usado quando o argumento está em uma linha, premissas separadas por vírgula.

a → b, a ⊢ b

Lê-se: "dados a→b e a, conclui-se b"
Vertical
∴ — formato empilhado
Usado quando as premissas ficam empilhadas com traço separando.

a → b   (P1)
a       (P2)
∴ b      (Q)

Como não confundir
FormatoSímbolo corretoExemplo
Uma linha só⊢a → b, a ⊢ b
Empilhado vertical∴P1 / P2 / — / ∴ Q

Argumentos Válidos e Falácias
Um argumento é válido quando a conclusão é necessariamente verdadeira sempre que as premissas forem verdadeiras.
Válido ✅
Modus Ponens
P2 é igual ao antecedente de P1.

a → b   (P1)
a       (P2) ← igual ao antecedente
∴ b      (Q) ✅
Funciona com negação! ~a → b com ~a na P2 = ainda é Modus Ponens.
Válido ✅
Modus Tollens
P2 é a negação do consequente de P1.

a → b    (P1)
~b       (P2) ← negação do consequente
∴ ~a      (Q) ✅
Se o efeito não aconteceu, a causa também não aconteceu.
Inválido ❌
Negação do Antecedente
P2 é a negação do antecedente de P1.

a → b    (P1)
~a       (P2) ← negação do antecedente
∴ ~b      (Q) ❌
Inválido ❌
Afirmação do Consequente
P2 é igual ao consequente de P1.

a → b    (P1)
b        (P2) ← igual ao consequente
∴ a       (Q) ❌
Macete da prova
Olha a P2 e pergunta: com o que ela se parece?

Igual ao antecedente de P1Modus Ponens✅ Válido

Prova Condicional
Usada quando a conclusão tem formato condicional (a → b).
Macete principal
A PP é o antecedente de Q!
Se a conclusão é c → ~a:

PP = c (o antecedente)
Objetivo = provar ~a (o consequente)

PP não bate com nenhuma premissa? Impossível — argumento mal formulado.
Macete
Por onde começo?
Olha o antecedente de cada premissa e verifica com o que bate:

Antecedente da premissaUsa com
Igual à PPDireto com PP
Igual a resultado anteriorCom esse resultado
Exemplo resolvido
a → b, c → ~b ⊢ c → ~a
Q é c → ~a, então PP = c.

Passo 1: a → b           Premissa 1
Passo 2: c → ~b          Premissa 2
Passo 3: c                Premissa Provisória (PP)
Passo 4: ~b               Modus Ponens (2, PP)
Passo 5: ~a               Modus Tollens (1, ~b)
Conclusão: c → ~a        Prova Condicional ✅
Redução ao Absurdo
Usada quando a conclusão não tem formato condicional. Nega a conclusão e infere até achar uma contradição.
Macete principal
A PP é a negação de Q!
Conclusão QPP
~(a ∧ c)a ∧ c
~(a ∧ ~c)a ∧ ~c
Quando Q é ~(algo), a PP é esse algo sem a negação!
Macete crucial
MP vs MT — não confunda!
SituaçãoRegraResultado
Tenho a→b e aModus PonensConclui b
Tenho a→b e ~bModus TollensConclui ~a
Exemplo resolvido
a → ~b, c → b ⊢ ~(a ∧ c)
Passo 1: a → ~b   Premissa 1
Passo 2: c → b    Premissa 2
Passo 3: a ∧ c    PP (negação de ~(a∧c))
Passo 4: a         Simplificação (PP)
Passo 5: c         Simplificação (PP)
Passo 6: ~b        Modus Ponens (1, a)
Passo 7: b         Modus Ponens (2, c)
Passo 8: b ∧ ~b   CONTRADIÇÃO! ❌
∴ ~(a ∧ c)       Redução ao Absurdo ✅

Questão 01 — 1 ponto
CPU em 12%, disco em 98% — qual é o fenômeno?
O sistema usa o disco como extensão da RAM (swap). Como o disco é milhares de vezes mais lento que a RAM, o desempenho despenca drasticamente.

✓ Resposta: B) Thrashing

Macete: CPU ociosa + disco 100% = sempre Thrashing. O SO está ocupado demais paginando para executar processos!

Questão 03 — Estados de processo

Questão 04 — Concorrência
Dois processos modificam a mesma conta sem sincronização
✓ Resposta: C) Race Condition — resolvido com mutex

O que aconteceu: A leu R$1.500 → B leu R$1.500 → A calculou R$1.800 → B calculou R$1.300 → B salvou R$1.300 → A sobrescreveu com R$1.800. O depósito de R$300 se perdeu!
ProblemaO que éSolução

StarvationProcesso nunca recebe a CPURound Robin
Macete: Race Condition = corrida pelos dados. Mutex (mutual exclusion) = trava o recurso enquanto um processo usa.
Questão 10 — Escalonamento
Principal vantagem do Round Robin sobre o FCFS
✓ Resposta: Menor tempo de resposta / Sem efeito comboio

AlgoritmoComo funcionaProblema
FCFSOrdem de chegada, sem preempçãoEfeito comboio — processo longo trava os curtos
Round RobinQuantum fixo por processo, preempção ao fimTempo de resposta justo para todos
Resposta completa: O Round Robin garante que nenhum processo monopolize a CPU — cada um recebe uma fatia de tempo (quantum) e volta ao final da fila, eliminando o efeito comboio e reduzindo o tempo de resposta médio.

Adicionar mais memória RAM

Q 08Qual o código octal da permissão rwxr-xr-x?

Q 09Converter permissão octal 644 para notação simbólica (rwx)

Q 10Principal vantagem do Round Robin sobre o FCFS — questão dissertativa
FCFS

First Come, First Served

Sem preempção — processo longo monopoliza CPU

Efeito comboio: processos curtos esperam atrás dos longos

Alto tempo de resposta médio

Round Robin

Cada processo recebe um quantum (fatia de tempo)

Ao esgotar o quantum, volta ao final da fila

Preempção garantida — ninguém monopoliza

Baixo tempo de resposta para todos

✎
Resposta dissertativa
Menor tempo de resposta / Sem efeito comboio
O Round Robin distribui a CPU de forma justa entre todos os processos usando um quantum fixo. Isso elimina o "efeito comboio" do FCFS, onde processos curtos ficam presos atrás de processos longos, resultando em menor tempo de resposta médio em sistemas de tempo compartilhado.
